# SentinelXDR — Database Schema

**Version:** 0.1.0 (Phase 0)
**Status:** Draft
**Last Updated:** 2026-06-09
**Database:** MongoDB 7.x

---

## 1. Design Principles

- All documents include `created_at` and `updated_at` timestamps (UTC ISO8601)
- All primary identifiers use `_id` (MongoDB ObjectId) with a human-readable `id` string alias (prefix + ULID, e.g., `alr_01HXYZ`)
- High-volume collections (`events`) use TTL indexes for automatic data expiration
- All queries are designed around defined indexes — no full collection scans in hot paths
- Sensitive fields (passwords, tokens) are never stored in plaintext

## 2. Database: `sentinelxdr`

---

### Collection: `events`

Normalized telemetry events ingested from agents and other collectors.

**Volume:** High — millions of documents per day. TTL-indexed for automatic pruning.

```json
{
  "_id": "ObjectId",
  "id": "evt_01HXYZ",
  "agent_id": "agt_01HXYZ",
  "asset_id": "ast_01HABC",
  "hostname": "WORKSTATION-01",
  "event_type": "process_create",
  "domain": "endpoint",
  "os": "linux",
  "timestamp": "ISODate",
  "received_at": "ISODate",
  "data": {
    "pid": 4521,
    "ppid": 1234,
    "process_name": "bash",
    "command_line": "bash -c 'whoami'",
    "user": "root",
    "executable_path": "/bin/bash",
    "md5": "abc123def456",
    "sha256": "abcdef1234567890"
  },
  "enrichment": {
    "geoip": {
      "country": "US",
      "city": "New York",
      "lat": 40.7128,
      "lon": -74.0060
    },
    "threat_intel_match": false,
    "ioc_ids": []
  },
  "created_at": "ISODate",
  "expires_at": "ISODate"
}
```

**Indexes:**
```
{ "asset_id": 1, "timestamp": -1 }          // Alert correlation queries
{ "event_type": 1, "timestamp": -1 }        // Rule engine event type filter
{ "data.pid": 1, "timestamp": -1 }          // Process tree pivot
{ "data.dst_ip": 1 }                        // Network IOC matching
{ "expires_at": 1 }  TTL: 0                 // Auto-delete after retention period
```

**Retention:** Configurable (default 90 days). Extended retention requires archiving to cold storage.

## 3. Redis Schema

### Streams

| Stream Key | Consumers | Description |
|---|---|---|
| `stream:events:raw` | Detection workers | Normalized events awaiting detection |
| `stream:alerts:new` | API WebSocket handler | New alerts for real-time push to frontend |
| `stream:commands:pending` | Agent command dispatcher | Queued agent commands |

### Hashes / Strings

| Key Pattern | Type | TTL | Description |
|---|---|---|---|
| `cache:asset:{id}` | Hash | 300s | Cached asset document |
| `cache:geoip:{ip}` | String (JSON) | 86400s | GeoIP lookup result |
| `session:{token_jti}` | String (JSON) | 28800s | User session data |
| `ratelimit:ingest:{agent_id}` | String (counter) | 60s | Sliding window rate limiter |
| `ratelimit:api:{user_id}` | String (counter) | 60s | API rate limiter per user |
| `agent:config:{agent_id}` | Hash | — | Agent configuration (no expiry) |

## 4. Data Retention Policy

| Collection | Default Retention | Notes |
|---|---|---|
| `events` | 90 days | TTL on `expires_at` field |
| `alerts` | 1 year | Manual review before deletion |
| `incidents` | 2 years | — |
| `assets` | Indefinite | Soft-delete on decommission |
| `audit_log` | 3 years | Legally may require longer; no delete permitted |
| `threat_intel` | Per IOC `expires_at` | Default indefinite |
